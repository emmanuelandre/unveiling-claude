# MultiCloud Dashboard

A unified multi-cloud management platform for managing resources across AWS, Azure, and Google Cloud Platform.

> **Training Exercise**: This project is designed as a hands-on implementation exercise using AI-assisted development tools (Claude Code, Cursor, Windsurf). Trainees will build a full-stack application from this specification.

## Project Overview

MultiCloud Dashboard provides a single pane of glass for managing cloud infrastructure across multiple providers. Users can:

- Connect and manage multiple cloud accounts (AWS, Azure, GCP)
- View and control virtual machines, storage, and networking resources
- Automate operations with scripts, templates, and schedules
- Enforce policies through rules and team-based access control

---

## Learning Objectives

By completing this project, trainees will learn:

1. **Backend Development**: Building RESTful APIs with Golang
2. **Database Design**: Modeling complex relationships in SQL
3. **Cloud Integration**: Working with multi-provider SDKs
4. **Frontend Development**: React with enterprise design systems
5. **Full-Stack Architecture**: Connecting all layers of a modern web application
6. **AI-Assisted Development**: Leveraging Claude Code for rapid development

## Features

### Navigation Menu

| Menu Item | Description |
|-----------|-------------|
| **Clouds** | Manage cloud provider accounts and credentials |
| **Machines** | List, start, stop, reboot virtual machines |
| **Volumes** | Block storage management (EBS, Azure Disks, Persistent Disks) |
| **Buckets** | Object storage (S3, Blob Storage, Cloud Storage) |
| **Networks** | VPCs, subnets, security groups |
| **Zones** | Availability zones and regions |
| **Keys** | SSH key management |
| **Images** | OS images and AMIs |
| **Scripts** | Automation scripts for provisioning |
| **Templates** | Deployment templates (CloudFormation, ARM, Terraform) |
| **Schedules** | Scheduled tasks (start/stop VMs, backups) |
| **Rules** | Policy enforcement and alerts |
| **Teams** | User groups and access control |

### Core Tables SQL

```sql
-- Clouds table
CREATE TABLE clouds (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    provider VARCHAR(50) NOT NULL CHECK (provider IN ('aws', 'azure', 'gcp')),
    credentials JSONB NOT NULL,
    region VARCHAR(100),
    owner_id UUID NOT NULL REFERENCES users(id),
    team_id UUID REFERENCES teams(id),
    enabled BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_synced_at TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Machines table
CREATE TABLE machines (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    cloud_id UUID NOT NULL REFERENCES clouds(id) ON DELETE CASCADE,
    external_id VARCHAR(255) NOT NULL,
    name VARCHAR(255) NOT NULL,
    state VARCHAR(50) NOT NULL,
    instance_type VARCHAR(100),
    public_ip INET,
    private_ip INET,
    zone_id UUID REFERENCES zones(id),
    image_id UUID REFERENCES images(id),
    tags JSONB DEFAULT '{}',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(cloud_id, external_id)
);

-- Volumes table
CREATE TABLE volumes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    cloud_id UUID NOT NULL REFERENCES clouds(id) ON DELETE CASCADE,
    machine_id UUID REFERENCES machines(id),
    external_id VARCHAR(255) NOT NULL,
    name VARCHAR(255),
    size_gb INTEGER NOT NULL,
    volume_type VARCHAR(50),
    state VARCHAR(50) NOT NULL,
    encrypted BOOLEAN DEFAULT false,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(cloud_id, external_id)
);
```

## Functional Requirements

### FR-1: Authentication & Authorization

| ID | Requirement |
|----|-------------|
| FR-1.1 | Users can register with email and password |
| FR-1.2 | Users can log in and receive JWT tokens |
| FR-1.3 | Tokens expire after 24 hours; refresh tokens last 7 days |
| FR-1.4 | Team members inherit permissions from team role (admin, member, viewer) |
| FR-1.5 | Resources are scoped to user or team ownership |

### FR-2: Cloud Management

| ID | Requirement |
|----|-------------|
| FR-2.1 | Users can add cloud providers (AWS, Azure, GCP) with credentials |
| FR-2.2 | Credentials are encrypted at rest |
| FR-2.3 | Users can test cloud connection before saving |
| FR-2.4 | Manual sync fetches all resources from cloud provider |
| FR-2.5 | Optional: Automatic sync runs on configurable interval |
| FR-2.6 | Users can enable/disable clouds without deleting them |

### FR-3: Machine Management

| ID | Requirement |
|----|-------------|
| FR-3.1 | List all machines across all connected clouds |
| FR-3.2 | Filter machines by cloud, state, instance type, tags |
| FR-3.3 | Search machines by name or IP address |
| FR-3.4 | Start, stop, and reboot machines |
| FR-3.5 | View machine details including monitoring metrics (if available) |
| FR-3.6 | Bulk actions: start/stop/reboot multiple selected machines |

### FR-4: Storage Management

| ID | Requirement |
|----|-------------|
| FR-4.1 | List all volumes with size, state, and attachment info |
| FR-4.2 | Attach volume to a machine |
| FR-4.3 | Detach volume from a machine |
| FR-4.4 | Create new volumes (provider-specific options) |
| FR-4.5 | List all buckets with size and object count |
| FR-4.6 | View bucket details (read-only, no object management required) |

### FR-5: Network & Infrastructure

| ID | Requirement |
|----|-------------|
| FR-5.1 | List all VPCs/VNets with CIDR blocks |
| FR-5.2 | View subnets and security groups per network |
| FR-5.3 | List availability zones per cloud |
| FR-5.4 | List available OS images per cloud |

### FR-6: SSH Keys

| ID | Requirement |
|----|-------------|
| FR-6.1 | Upload SSH public keys |
| FR-6.2 | Generate new SSH key pairs (private key shown once) |
| FR-6.3 | Associate keys with clouds for provisioning |
| FR-6.4 | Delete keys |

### FR-7: Automation

| ID | Requirement |
|----|-------------|
| FR-7.1 | Create, edit, delete shell scripts |
| FR-7.2 | Run scripts on selected machines (optional SSH execution) |
| FR-7.3 | Create deployment templates (store as text) |
| FR-7.4 | Create schedules with cron expressions |
| FR-7.5 | Schedules can target machines for start/stop actions |
| FR-7.6 | View schedule execution history |

### FR-8: Rules & Policies

| ID | Requirement |
|----|-------------|
| FR-8.1 | Create rules with conditions (e.g., "CPU > 80%") |
| FR-8.2 | Define actions for rules (e.g., "send notification", "stop machine") |
| FR-8.3 | Enable/disable rules |
| FR-8.4 | Rules are evaluated during sync (not real-time required) |

### FR-9: Team Management

| ID | Requirement |
|----|-------------|
| FR-9.1 | Create teams with name and description |
| FR-9.2 | Invite users to teams by email |
| FR-9.3 | Assign roles: admin, member, viewer |
| FR-9.4 | Share clouds with teams |
| FR-9.5 | Remove members from teams |

## Implementation Phases

### Phase 1: Foundation
**Goal:** Set up project structure and basic infrastructure

- [ ] Initialize Go module with Gin framework
- [ ] Set up PostgreSQL database and migrations
- [ ] Implement user authentication (register, login, JWT)
- [ ] Create React app with Dell Design System
- [ ] Build basic layout (sidebar, header, routing)
- [ ] Connect frontend to backend auth endpoints

### Phase 2: Core CRUD Operations
**Goal:** Implement core resource management

- [ ] Clouds: full CRUD with credential storage
- [ ] Implement cloud provider interface (abstract)
- [ ] Machines: list, detail view (mock data initially)
- [ ] Volumes: list and detail view
- [ ] Buckets, Networks, Zones: read-only listing
- [ ] Keys: CRUD operations
- [ ] Build reusable DataTable component

### Phase 3: Cloud Integration
**Goal:** Connect to real cloud providers

- [ ] AWS provider: EC2, EBS, S3, VPC integration
- [ ] Azure provider: VMs, Disks, Blob, VNet integration
- [ ] GCP provider: Compute, Disks, Storage, VPC integration
- [ ] Implement sync functionality
- [ ] Machine actions: start, stop, reboot
- [ ] Volume actions: attach, detach

### Phase 4: Advanced Features
**Goal:** Automation and team features

- [ ] Scripts: CRUD and basic execution
- [ ] Templates: CRUD (storage only)
- [ ] Schedules: CRUD with cron parsing
- [ ] Rules: CRUD and basic evaluation
- [ ] Teams: CRUD and member management
- [ ] Sharing resources with teams

### Phase 5: Polish & Production Readiness
**Goal:** Finalize for production use

- [ ] Search and filtering across all resources
- [ ] Bulk actions for machines
- [ ] Error handling and user feedback
- [ ] Loading states and optimistic updates
- [ ] API documentation (Swagger/OpenAPI)
- [ ] Docker Compose for local development
- [ ] Basic unit tests for critical paths

## Getting Started

### Prerequisites

- Go 1.21+
- Node.js 18+
- PostgreSQL 15+
- Docker (optional, for containerized development)

### Environment Variables

```bash
# Backend (.env)
DATABASE_URL=postgres://user:pass@localhost:5432/multicloud?sslmode=disable
JWT_SECRET=your-secret-key-min-32-chars
PORT=8080

# Cloud Provider Credentials (for testing)
AWS_ACCESS_KEY_ID=...
AWS_SECRET_ACCESS_KEY=...
AZURE_TENANT_ID=...
AZURE_CLIENT_ID=...
AZURE_CLIENT_SECRET=...
GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json
```

### Running Locally

```bash
# 1. Clone the repository
git clone <repository-url>
cd multicloud-dashboard

# 2. Start PostgreSQL (Docker)
docker run -d --name postgres \
  -e POSTGRES_USER=user \
  -e POSTGRES_PASSWORD=pass \
  -e POSTGRES_DB=multicloud \
  -p 5432:5432 \
  postgres:15

# 3. Run migrations
cd backend
make migrate-up

# 4. Start backend
make run

# 5. Start frontend (new terminal)
cd frontend
npm install
npm run dev
```

### Development Commands

```bash
# Backend
make build          # Build binary
make run            # Run server
make test           # Run tests
make migrate-up     # Apply migrations
make migrate-down   # Rollback migrations
make swagger        # Generate API docs

# Frontend
npm run dev         # Development server
npm run build       # Production build
npm run test        # Run tests
npm run lint        # Lint code
```
